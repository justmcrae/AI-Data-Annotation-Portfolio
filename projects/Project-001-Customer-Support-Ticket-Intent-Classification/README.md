# Project 001 – Customer Support Ticket Intent Classification

## Overview

This project demonstrates a text classification workflow for customer support tickets.

The objective was to classify each customer message into its primary intent using predefined annotation guidelines. 
The project simulates a real-world AI data annotation task commonly used to train customer support chatbots, ticket 
routing systems, and natural language processing (NLP) models.

---

## Objective

To accurately assign one intent label to each customer support message while maintaining consistency, documenting ambiguous 
cases, and performing quality assurance (QA) review.

---

## Dataset

- **Dataset Size:** 100 customer support messages
- **Language:** English
- **Task Type:** Multi-class Text Classification
- **Format:** Microsoft Excel (.xlsx)

---

## Label Set

The following labels were used:

| Label           | Description                                                                        |
|-----------------|------------------------------------------------------------------------------------|
| Billing         | Payment issues, refunds, invoices, subscriptions, charges                          |
| Account         | Login, passwords, account settings, verification, profile management               |
| Shipping        | Delivery status, tracking, missing or damaged orders                               |
| Technical Issue | Application errors, bugs, crashes, loading failures, system malfunctions           |
| General Inquiry | Product information, policies, pricing, features, and other informational requests |

---

## Annotation Process

The annotation workflow included:

1. Reading each customer message carefully.
2. Identifying the customer's primary intent.
3. Assigning the most appropriate label.
4. Recording notes for ambiguous cases when necessary.
5. Reviewing annotations through a QA process.
6. Correcting inconsistent labels identified during review.

---

## Quality Assurance

Quality assurance focused on:

- Label consistency
- Correct identification of the primary intent
- Resolution of ambiguous cases
- Documentation of annotation decisions

Reviewer feedback was incorporated to improve annotation quality and consistency.
