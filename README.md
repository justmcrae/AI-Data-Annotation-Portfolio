## Annotation Workflow

Each project follows a structured workflow:

1. Review the annotation guidelines.
2. Read each data sample carefully.
3. Identify the primary annotation target.
4. Assign the appropriate label.
5. Document ambiguous cases when necessary.
6. Perform quality assurance review.
7. Apply corrections and finalize the dataset.
8. Publish project documentation.
